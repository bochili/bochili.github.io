黑苹果安装Big Sur过程中遇到的问题总结

发布于 2020-07-01 解决问题  Mac 

Windows 下 VMWare WorkStation Pro 中安装程序报 BIErrorDomain 错误 3

这是因为在 macOS 下，VMWare 直接读取系统的 smbios 信息，而在 Windows 中无法读取，所以要在虚拟机目录下的 vmx 文件中添加 smbios 信息：

示例如下：

smbios.reflectHost = "TRUE"
hw.model = "iMac15,1"
board-id = "Mac-42FD25EABCABB274"

该思路来自于https://www.youtube.com/watch?v=fOekfmgMM00

Board-ID 机型对照表：http://bbs.pcbeta.com/viewthread-1718853-1-1.html

macOS 11 中开启 HiDPI 的问题

由于 macOS11 对于安全性更强的要求，macOS Catalina 中解锁 sle 的命令已经失效了。

参考此篇文章：http://bbs.pcbeta.com/viewthread-1862148-1-1.html

将 15 中System/Library下的 Displays 目录拷贝至 11 系统中的Library目录下，重启，使用软件“RDM”调整分辨率即可。

但是在我的电脑上，开启 HIDPI 之后，登录界面出现了一些比例问题，不影响使用但是很难受。

菜单栏上系统的一些图标卡顿

这个问题初步推断是无线的问题。因为没有开启 wifi，所以系统一直在尝试开启，导致卡顿。

而配备了无线网卡的 Hackintosh 就没有出现卡顿的情况。

建议

本来以为上 BigSur 会困难重重，然后出现一堆问题，但是我是真的没想到用 bat 大佬的 vm 大法竟然一次成功了，没有遇到什么问题，但是目前来讲 BigSur 问题是挺多的，包括应用的适配、系统自身的 bug 等等等等，还是不推荐日用。

然后就是也不要想一口吃成个胖次，可能我只是运气比较好，撞上了。几位朋友使用我的 EFI 依然会出现卡 EB 各种问题。

总而言之，BigSur 相比于之前的 OS X 系列，变化还是挺大的，尝鲜有风险，切勿盲目更新。
